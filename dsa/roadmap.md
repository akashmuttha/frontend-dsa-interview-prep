# DSA Roadmap

This roadmap follows the DSA master system: build pattern recognition first, then data structures, then advanced problem solving, then interview readiness.

## Principle

The goal is not to finish random questions.

The goal is to recognize patterns, explain the approach clearly, and write correct code under interview conditions.

## Part 1: Foundations

Folder: `dsa/00-foundations/`

Focus areas:

- Big O and complexity thinking
- Arrays
- Hash maps and sets
- Frequency counters
- Prefix sum
- Two pointers
- Sliding window
- Binary search
- Recursion basics

Expected output:

- Understand the trigger for each pattern
- Solve foundation problems in Python
- Write short notes explaining why the pattern applies
- Re-solve important problems without looking

## Part 2: Core Data Structures

Folder: `dsa/01-data-structures/`

Focus areas:

- Linked lists
- Stacks and queues
- Binary trees and BSTs
- DFS and BFS
- Heaps
- Intervals
- Greedy thinking
- Graph traversal
- Backtracking

Expected output:

- Implement traversals confidently
- Know when to use DFS vs BFS
- Recognize heap, interval, graph, and backtracking triggers
- Keep notes on mistakes and edge cases

## Part 3: Advanced Problem Solving

Folder: `dsa/02-dp-backtracking/`

Focus areas:

- Dynamic programming foundations
- Memoization
- Tabulation
- 1D DP
- Grid DP
- Subsequence DP
- Knapsack-style thinking
- Mixed pattern recognition
- Weak area repair

Expected output:

- Define state clearly
- Write recurrence before coding
- Explain memoization and tabulation tradeoffs
- Practice mixed problems without relying on labels

## Part 4: Interview Readiness

Folder: `dsa/03-interview-revision/`

Focus areas:

- Pattern consolidation
- Timed problem solving
- Mock interview practice
- Communication under pressure
- Final revision list

Core patterns to revise:

- Hash maps
- Sets
- Frequency counters
- Two pointers
- Sliding window
- Prefix sum
- Linked lists
- Fast and slow pointers
- Binary search
- DFS
- BFS
- Heaps
- Greedy
- Graphs
- Backtracking
- Dynamic programming

Expected output:

- Explain the pattern before coding
- State complexity clearly
- Handle edge cases out loud
- Track weak patterns and revisit them

## Problem Folder Standard

Each polished problem should use this structure:

```text
problem-name/
  problem.md
  solution.py
  notes.md
```

Optional frontend-language rewrites can be added when useful:

```text
solution.js
solution.ts
```

## Notes Standard

Each `notes.md` should answer:

- What pattern is this?
- What is the mental trigger?
- What is the approach?
- What are the edge cases?
- What is the time and space complexity?
- What mistake should I avoid next time?
