# Frontend Roadmap

This roadmap combines JavaScript, TypeScript and React practice into one frontend engineering track.

The goal is to build strong fundamentals, reusable UI thinking, and interview-ready implementation skill.

## Principle

Do not only watch course videos.

For each topic:

1. Recall or implement from memory.
2. Check with examples and edge cases.
3. Rewatch only the exact weak concept.
4. Rebuild later without the lecture.

## Track 1: JavaScript

Folder: `frontend/00-javascript/`

Focus areas:

- Values, types, variables and functions
- Arrays, objects, references and copying
- Scope, closures, hoisting and execution context
- `this`, arrows, `call`, `apply`, `bind`, constructors and prototypes
- Promises, event loop, async/await and error handling
- Debounce, throttle, memoization, cloning and flattening
- DOM, events, delegation, storage and modules
- Browser, networking and frontend security fundamentals
- Output questions and implementation utilities

Expected outputs:

- Small concept notes
- Utility implementations from scratch
- Output-question explanations
- Browser/networking explanations
- Dataset transformation exercises

## Track 2: TypeScript

Folder: `frontend/01-typescript/`

Focus areas:

- Primitives, inference, functions, objects and arrays
- Unions, literals, narrowing and interfaces
- React props, state, events, refs and reusable APIs
- Discriminated unions for async UI states
- Generics for helpers, hooks and reusable components
- Domain models, API boundaries and module contracts
- Utility types, `keyof`, mapped types and safe error handling
- Strict typing for machine coding and project work

Expected outputs:

- Typed JavaScript exercises
- Typed React component examples
- Typed custom hooks
- Typed API and UI-state models
- Strict machine-coding implementations without unnecessary `any`

## Track 3: React

Folder: `frontend/02-react/`

Focus areas:

- Components, props, state and list rendering
- Controlled forms, derived state and lifting state
- Composition, reusable component APIs and render reasoning
- Effects, cleanup, async states and request cancellation
- Refs, custom hooks, reducers and context
- Routing, layouts, URL state and data flows
- Performance measurement and optimization
- Accessibility, focus management and testing
- Error boundaries, protected routes and reliability

Expected outputs:

- Typed component exercises
- Reusable component patterns
- Custom hooks
- Data-fetching screens with loading, empty and error states
- Performance and accessibility notes

## Track 4: Machine Coding

Folder: `frontend/03-machine-coding/`

Target exercises:

- Todo app
- Autocomplete
- Pagination
- Modal system
- Data table
- File explorer or nested comments

Each exercise should include:

```text
README.md
src/
package.json
```

The README should explain features, state design, edge cases, accessibility notes and possible improvements.

## Track 5: Projects

Folder: `frontend/04-projects/`

Projects should connect JavaScript, TypeScript and React fundamentals into real UI work.

Each project should document:

- Goal
- Features
- Tech stack
- Setup
- Architecture notes
- Tradeoffs
- Improvements
