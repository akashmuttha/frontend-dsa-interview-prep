# React Track

React practice is TypeScript-first and focused on building reusable, understandable UI.

## Focus Areas

- Components, JSX and props
- State, events and list rendering
- Controlled forms and derived state
- Lifting state and immutable updates
- Composition and reusable component APIs
- Render, commit, keys and re-render reasoning
- Effects, cleanup and async data states
- Refs, custom hooks, reducers and context
- Routing, layouts and URL state
- Performance measurement and optimization
- Accessibility, focus management and testing
- Error boundaries and reliability

## Practice Areas

- `hooks/` for custom hooks and hook behavior
- `component-patterns/` for reusable component APIs
- `state-management/` for reducer/context/state-placement exercises

## Output Standard

Each React exercise should include:

- typed props and state
- clear component boundaries
- loading, empty and error states when async
- notes on re-render behavior where relevant
- accessibility considerations for interactive UI

## Notes Standard

Each React note should answer:

- What component responsibility is being modeled?
- Where does state live and why?
- What causes re-renders?
- What edge cases exist?
- How is the component typed?
