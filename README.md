# Frontend + DSA Practice

A structured repository for frontend engineering practice, DSA problem solving, and interview preparation.

This repo is organized to show both code and thinking: solutions, pattern notes, frontend concepts, machine coding exercises, and project work.

## Repository Map

```text
dsa/
  00-foundations/          Core patterns: arrays, hash maps, two pointers, sliding window
  01-data-structures/      Linked lists, stacks, queues, trees, heaps, graphs
  02-dp-backtracking/      Dynamic programming and backtracking practice
  03-interview-revision/   Pattern summaries, timed practice, mock notes
  leetcode-inbox/          New problems generated from the LeetCode extension

frontend/
  00-javascript/           JavaScript concepts and practice
  01-typescript/           TypeScript concepts and practice
  02-react/                Hooks, component patterns, and state management
  03-machine-coding/       Frontend interview-style coding exercises
  04-projects/             Larger frontend projects

notes/                     Learning notes and weekly reviews
templates/                 Reusable templates for problems and projects
archive/                   Older or unpolished files kept out of the main flow
```

## DSA

DSA work is grouped by pattern or topic instead of being kept as one long list of files.

Each polished problem folder follows this shape:

```text
problem-name/
  problem.md
  solution.py
  notes.md
```

Primary DSA language: Python.

Selected problems may also include JavaScript or TypeScript versions when useful for frontend interview preparation.

## Frontend

Frontend work is split into JavaScript, TypeScript, React, machine coding exercises, and larger projects.

See `frontend/roadmap.md` for the frontend learning flow.

Machine coding and project folders will include:

```text
README.md
src/
package.json
```

Each project README should explain the goal, features, setup, implementation notes, and possible improvements.
